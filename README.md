# Eurovision voting bias — Charron (2013) replication & extension

Replication of Charron (2013), *"Impartiality, friendship-networks and voting
behavior: Evidence from voting patterns in the Eurovision Song Contest"*
(*Social Networks* 35: 484–497), plus an extension of the analysis through 2026.

Central finding (H1): **impartial countries temper their friend-voting bias** —
the Friend × Impartiality interaction is negative. It replicates, and it largely
holds through 2026.

## Layout
- **`replication/`** — faithful Charron 1975–2012 replication (Tables 1–4,
  Appendix 3, Figs 1–3). Python builds the data; **R (`AER::tobit` / `censReg`) is
  the authoritative estimator**. Run: `python3 replication/run_all.py`.
- **`extension/`** — everything beyond the paper: the eschome.net scrapers, the
  1975–2026 extended panel, voting-channel (jury vs public) and Rest-of-World
  analyses. Run: `python3 extension/run_all_extension.py`.
- **`slides/`** — Beamer decks (`charron_vs_replication.pdf`, `extension.pdf`).
  Compile **twice** (`pdflatex`) for the `x/y` page footer.
- Per-folder `SUMMARY.md` files log the full history of prompts and results.

## Requirements
- Python 3 with `pandas`, `numpy`, `scipy`, `statsmodels`, `matplotlib`.
- R with `AER`, `sandwich`, `lmtest`, `censReg`, `plm` (for the authoritative
  regression tables). The pipeline runs without R but skips the `*_R.txt` tables.

## Data note — the QoG / ICRG file
Impartiality is the ICRG "Quality of Government" score (`icrg_qog`) from the
**QoG Standard Time-Series**. The full release is a 60–70 MB, 1,800-column CSV,
so it is **not committed**. The pipeline reads only three columns
(`ccodealp`, `year`, `icrg_qog`), which are committed as the slim subset
**`qog_icrg_jan26.csv`** (~150 KB) — so everything runs out of the box.

To refresh with a newer QoG release:
1. Download the QoG Standard Time-Series CSV from
   <https://www.gu.se/en/quality-government/qog-data/data-downloads/standard-dataset>.
2. Subset it (any tool), e.g.:
   ```python
   import pandas as pd
   pd.read_csv("qog_std_ts_janXX.csv", usecols=["ccodealp","year","icrg_qog"]) \
     .dropna(subset=["icrg_qog"]).to_csv("qog_icrg_jan26.csv", index=False)
   ```
3. The full `qog_std_ts_*.csv` files are `.gitignore`d.

Other large/derived files kept out of git (regenerated by the `run_all` scripts):
the eschome scraper cache and the multi-MB `master*.csv` / `regression_data*.csv`
panels. Scraped source data (`extension/output/votes_eschome.csv`,
`finalists_eschome.csv`) **is** committed so the pipeline runs without re-scraping.
